```matlab
function plotWindBarb(X,Y,speed,dir_from,varargin)
% plotWindBarb Draw simple meteorological wind barbs at X,Y
%   X,Y,speed,dir_from are vectors of same length.
%   dir_from: degrees, meteorological (wind coming from).
%   Optional name-value: 'Scale' (length scale), 'Color', 'LineWidth'.

% Parse options
p = inputParser; addParameter(p,'Scale',0.04,@isnumeric);
addParameter(p,'Color','k');
addParameter(p,'LineWidth',1);
parse(p,varargin{:});
S = p.Results.Scale; C = p.Results.Color; LW = p.Results.LineWidth;

% Barb geometry parameters (fraction of scale)
shaft = [0 1];         % line from (0,0) to (0,1)
flag_len = 0.25;       % triangle flag length (for 50 kt)
feather_len = 0.2;     % single feather length (for 10 kt)
gap = 0.02;

n = numel(X);
holdState = ishold; hold on
for i=1:n
    spd = speed(i);
    if spd < 2, continue; end  % skip near calm
    % Determine number of 50, 10, 5 knots
    n50 = floor(spd/50); spd = spd - 50*n50;
    n10 = floor(spd/10); spd = spd - 10*n10;
    n5  = round(spd/5);  % remaining 5-9 -> one 5 knot
    % Build barb segments along shaft (from tip backward)
    pos = 1;  % start at tip (shaft length = 1)
    segs = {}; % store line segments and filled flags
    for k=1:n50
        % triangle flag
        base = pos; tip = pos - flag_len;
        % triangle coordinates in local shaft coords (x,y)
        tri = [0,base;  -flag_len,base; 0,tip];
        segs{end+1} = struct('type','patch','xy',tri); %#ok<*AGROW>
        pos = pos - flag_len - gap;
    end
    for k=1:n10
        % long feather (line)
        xy = [0,pos; -feather_len,pos+feather_len];
        segs{end+1} = struct('type','line','xy',xy);
        pos = pos - feather_len - gap;
    end
    if n5==1
        xy = [0,pos; -feather_len/2,pos+feather_len/2];
        segs{end+1} = struct('type','line','xy',xy);
        pos = pos - feather_len/2 - gap;
    end
    % Draw shaft and elements rotated by wind direction
    theta = deg2rad(270 - dir_from(i)); % convert meteorological to math angle
    R = [cos(theta) -sin(theta); sin(theta) cos(theta)];
    % shaft
    p1 = [0,0]*S; p2 = [0,1]*S;
    P = [p1; p2]*R;
    plot(X(i)+P(:,1), Y(i)+P(:,2), 'Color', C, 'LineWidth', LW)
    % elements
    for s=1:numel(segs)
        xy = segs{s}.xy * S;
        xy = (xy * R) + [X(i), Y(i)];
        if strcmp(segs{s}.type,'line')
            plot(xy(:,1), xy(:,2), 'Color', C, 'LineWidth', LW)
        else
            patch(xy(:,1), xy(:,2), C, 'EdgeColor',C)
        end
    end
end
if ~holdState, hold off, end
axis equal
end
```

Copilot:
### Rotation formula (body → earth)

Use intrinsic (body-fixed) rotations applied in order: roll (φ) about x, pitch (θ) about y, heading/yaw (ψ) about z. The direction-cosine matrix (DCM) that maps a vector in body coordinates to earth coordinates is

R = Rz(ψ) * Ry(θ) * Rx(φ)

where

Rx(φ) = [1    0         0
         0  cosφ   -sinφ
         0  sinφ    cosφ]

Ry(θ) = [ cosθ   0  sinθ
            0    1    0
         -sinθ   0  cosθ]

Rz(ψ) = [cosψ  -sinψ   0
         sinψ   cosψ   0
           0      0    1]

Then earth_vec = R * body_vec.

Note on axis signs: you stated body axes are (forward, right, down) and earth axes are (east, north, up). Many aerospace DCM conventions assume body z is down and earth z is north/up depending on target frame (NED vs ENU). The formulas above are general; if you want ENU (east,north,up) and your body vector uses down-positive, you must negate the down component before or after transform (or convert NED ↔ ENU). See the MATLAB example below that handles FRD → ENU.

### MATLAB function (FRD → ENU)

```matlab
function enu = frd2enu(body, roll, pitch, heading, deg)
% frd2enu Convert vectors from aircraft FRD (forward,right,down) to
%           earth ENU (east,north,up).
%   enu = frd2enu(body, roll, pitch, heading)
%   body  is 3xN or Nx3 with components [forward; right; down]
%   roll, pitch, heading are scalars (or 1xN) in radians by default
%   enu = frd2enu(..., true) treats angles as degrees
%
% Example:
%   b = [10; 0; 0]; % 10 units forward
%   enu = frd2enu(b, 5, 2, 30, true);

if nargin<5, deg = false; end

% Ensure column vectors
if size(body,1)==3
    B = body;
else
    B = body.';
end

if deg
    roll = deg2rad(roll); pitch = deg2rad(pitch); heading = deg2rad(heading);
end

% Rotation matrices (intrinsic rotations: roll about x, pitch about y, heading about z)
cr = cos(roll); sr = sin(roll);
ct = cos(pitch); st = sin(pitch);
cp = cos(heading); sp = sin(heading);

Rx = [1  0   0;
      0 cr -sr;
      0 sr  cr];

Ry = [ ct  0  st;
       0   1   0;
      -st  0  ct];

Rz = [ cp -sp  0;
       sp  cp  0;
        0   0  1];

R = Rz * Ry * Rx;    % DCM body -> earth in same handedness

% Body uses down-positive; convert to a right-handed body z-up for rotation:
% either invert body z before rotation, or invert the resulting earth z after.
% Here we invert body z before rotating.
B(3,:) = -B(3,:);

E = R * B;           % E is [east; north; up] because we constructed R for x->east, y->north, z->up

% Return same orientation as input
enu = E;
end
```

Example usage:

```matlab
% Single vector: forward=100, right=10, down=2 (units)
body = [100; 10; 2];
φ = 5; θ = -2; ψ = 45;    % degrees
enu = frd2enu(body, φ, θ, ψ, true)
% enu = [east; north; up]
```

If your body vectors are given as velocities [u_forward; v_right; w_down], the same function applies.

### Vectorized frd2enu (no loop)

This version accepts body as 3xN or Nx3 (columns = [forward; right; down]) and roll, pitch, heading as scalars or 1xN / Nx1 vectors. Angles are radians by default; set deg=true if degrees.

```matlab
function enu = frd2enu(body, roll, pitch, heading, deg)
% frd2enu Vectorized FRD -> ENU without a for-loop
% enu = frd2enu(body, roll, pitch, heading)
% body: 3xN or Nx3 (columns are [forward; right; down])
% roll,pitch,heading: scalar or length-N vectors
% deg: optional logical (default false). If true, angles are degrees.
%
% Output enu is 3xN with rows [east; north; up].

if nargin<5, deg = false; end

% Normalize body to 3xN
if size(body,1)==3
    B = body;
elseif size(body,2)==3
    B = body.';
else
    error('body must be 3xN or Nx3');
end
N = size(B,2);

% Expand angles to 1xN
if isscalar(roll),  roll = repmat(roll,1,N); end
if isscalar(pitch), pitch = repmat(pitch,1,N); end
if isscalar(heading), heading = repmat(heading,1,N); end

roll   = reshape(roll,1,[]);
pitch  = reshape(pitch,1,[]);
heading= reshape(heading,1,[]);

if any([numel(roll), numel(pitch), numel(heading)] ~= N)
    error('roll, pitch, heading must be scalars or length-N vectors');
end

if deg
    roll = deg2rad(roll); pitch = deg2rad(pitch); heading = deg2rad(heading);
end

% Convert body down-positive to z-up for math rotations
B(3,:) = -B(3,:);

% Precompute trig functions (1xN)
cr = cos(roll); sr = sin(roll);
ct = cos(pitch); st = sin(pitch);
cp = cos(heading); sp = sin(heading);

% Compute DCM components for R = Rz(psi)*Ry(theta)*Rx(phi)
% r11 ... r33 are 1xN vectors
r11 =  cp.*ct;
r12 =  cp.*sr.*st - sp.*cr;
r13 =  cp.*cr.*st + sp.*sr;

r21 =  sp.*ct;
r22 =  sp.*sr.*st + cp.*cr;
r23 =  sp.*cr.*st - cp.*sr;

r31 = -st;
r32 =  ct.*sr;
r33 =  ct.*cr;

% Apply R to each column of B using elementwise operations
Bx = B(1,:); By = B(2,:); Bz = B(3,:);

E_x = r11.*Bx + r12.*By + r13.*Bz;   % east
E_y = r21.*Bx + r22.*By + r23.*Bz;   % north
E_z = r31.*Bx + r32.*By + r33.*Bz;   % up

enu = [E_x; E_y; E_z];
end
```

### Vectorized enu2frd (no loop)

This function converts ENU (east,north,up) vectors to FRD (forward,right,down) for the same intrinsic rotation convention:
R = Rz(ψ)*Ry(θ)*Rx(φ) with φ=roll, θ=pitch, ψ=heading, and body z positive down. Angles are radians by default; set deg=true if you pass degrees.

```matlab
function frd = enu2frd(enu, roll, pitch, heading, deg)
% enu2frd Vectorized ENU -> FRD without a for-loop
% frd = enu2frd(enu, roll, pitch, heading)
% enu: 3xN or Nx3 (columns are [east; north; up])
% roll,pitch,heading: scalar or length-N vectors (same order as frd2enu)
% deg: optional logical (default false). If true, angles are degrees.
%
% Output frd is 3xN with rows [forward; right; down].

if nargin<5, deg = false; end

% Normalize enu to 3xN
if size(enu,1)==3
    E = enu;
elseif size(enu,2)==3
    E = enu.';
else
    error('enu must be 3xN or Nx3');
end
N = size(E,2);

% Expand angles to 1xN
if isscalar(roll),  roll = repmat(roll,1,N); end
if isscalar(pitch), pitch = repmat(pitch,1,N); end
if isscalar(heading), heading = repmat(heading,1,N); end

roll   = reshape(roll,1,[]);
pitch  = reshape(pitch,1,[]);
heading= reshape(heading,1,[]);

if any([numel(roll), numel(pitch), numel(heading)] ~= N)
    error('roll, pitch, heading must be scalars or length-N vectors');
end

if deg
    roll = deg2rad(roll); pitch = deg2rad(pitch); heading = deg2rad(heading);
end

% Precompute trig (1xN)
cr = cos(roll); sr = sin(roll);
ct = cos(pitch); st = sin(pitch);
cp = cos(heading); sp = sin(heading);

% DCM components R = Rz*Ry*Rx as in frd2enu
r11 =  cp.*ct;
r12 =  cp.*sr.*st - sp.*cr;
r13 =  cp.*cr.*st + sp.*sr;

r21 =  sp.*ct;
r22 =  sp.*sr.*st + cp.*cr;
r23 =  sp.*cr.*st - cp.*sr;

r31 = -st;
r32 =  ct.*sr;
r33 =  ct.*cr;

% Apply R^T (which maps ENU -> body_z_up coordinates)
Ex = E(1,:); Ey = E(2,:); Ez = E(3,:);

% R^T * E  => body_z_up components (x_b_up; y_b_up; z_b_up)
bx_up = r11.*Ex + r21.*Ey + r31.*Ez;
by_up = r12.*Ex + r22.*Ey + r32.*Ez;
bz_up = r13.*Ex + r23.*Ey + r33.*Ez;

% Convert body z-up to body down-positive (FRD): invert z
bz_down = -bz_up;

frd = [bx_up; by_up; bz_down];
end
```

You need to convert your True Heading (θ_true, measured clockwise from north, as is usual in navigation) into the mathematical yaw angle ψ used in the Rz(ψ) matrix (which is CCW from the +x axis = east). The correct conversion is

ψ = 90° − θ_true

(or in radians ψ = pi/2 − θ_true_rad).

Reasoning (brief):
- Rz(ψ) in the standard math form Rz = [cos −sin; sin cos] uses ψ measured CCW from +x (east).
- True heading is normally measured clockwise from +y (north).
- Converting clockwise-from-north to CCW-from-east gives ψ = 90° − θ_true.

Example consequences:
- θ_true = 0° (north) → ψ = 90° → body-forward maps to +north (0,1).
- θ_true = 90° (east) → ψ = 0° → body-forward maps to +east (1,0).
- θ_true = 180° (south) → ψ = −90° → body-forward maps to −north (0,−1).

Short MATLAB snippet (degrees):

```matlab
% TrueHeading in degrees (navigation: 0=north, 90=east)
psi = deg2rad(90 - TrueHeading);   % yaw for Rz

cp = cos(psi); sp = sin(psi);
Rz = [cp -sp 0; sp cp 0; 0 0 1];
% then R = Rz * Ry * Rx as before
```

If your current code used psi = TrueHeading - 90 and gave forward→east, right→south, that indicates the sign/offset is reversed; switching to psi = 90 − TrueHeading will give forward→north and right→east (and, combined with the rest of the DCM and the down→up sign handling, produce ENU: east, north, up).

    phi   = deg2rad(roll);
    theta = deg2rad(pitch);
    H     = deg2rad(Heading);
else

ct = cos(theta); st = sin(theta);
cp = cos(psi); sp = sin(psi);

R = [ cp*ct,  cp*sr*st - sp*cr,  cp*cr*st + sp*sr;
      sp*ct,  sp*sr*st + cp*cr,  sp*cr*st - cp*sr;
      -st,    ct*sr,             ct*cr ];

b(3) = -b(3);

### frd2enu Using Geodetic Heading (vectorized, no loop)

This function treats Heading as geodetic/navigation heading (0° = north, 90° = east, measured clockwise). It converts Heading to the mathematical yaw ψ = 90° − Heading (ψ in radians) and uses the intrinsic rotation sequence Rx(roll) → Ry(pitch) → Rz(ψ) so R = Rz(ψ)*Ry(pitch)*Rx(roll). Input body is FRD (forward,right,down). Output is ENU (east,north,up).

```matlab
function enu = frd2enu(body, roll, pitch, heading, deg)
% frd2enu Convert FRD -> ENU using geodetic Heading (vectorized)
% enu = frd2enu(body, roll, pitch, heading)
% body: 3xN or Nx3 (columns are [forward; right; down])
% roll,pitch,heading: scalar or length-N vectors
% deg: optional logical (default false). If true, angles are degrees.
% Output enu is 3xN with rows [east; north; up].

if nargin<5, deg = false; end

% Normalize body to 3xN
if size(body,1)==3
    B = body;
elseif size(body,2)==3
    B = body.';
else
    error('body must be 3xN or Nx3');
end
N = size(B,2);

% Expand angles to 1xN
if isscalar(roll),   roll = repmat(roll,1,N); end
if isscalar(pitch),  pitch = repmat(pitch,1,N); end
if isscalar(heading),heading = repmat(heading,1,N); end

roll    = reshape(roll,1,[]);
pitch   = reshape(pitch,1,[]);
heading = reshape(heading,1,[]);

if any([numel(roll), numel(pitch), numel(heading)] ~= N)
    error('roll, pitch, heading must be scalars or length-N vectors');
end

if deg
    roll   = deg2rad(roll);
    pitch  = deg2rad(pitch);
    heading= deg2rad(heading);
end

% Convert geodetic Heading -> math yaw psi = 90deg - Heading
psi = pi/2 - heading;

% Precompute trig (1xN)
cr = cos(roll); sr = sin(roll);
ct = cos(pitch); st = sin(pitch);
cp = cos(psi); sp = sin(psi);

% DCM components for R = Rz(psi)*Ry(pitch)*Rx(roll)
r11 =  cp.*ct;
r12 =  cp.*sr.*st - sp.*cr;
r13 =  cp.*cr.*st + sp.*sr;

r21 =  sp.*ct;
r22 =  sp.*sr.*st + cp.*cr;
r23 =  sp.*cr.*st - cp.*sr;

r31 = -st;
r32 =  ct.*sr;
r33 =  ct.*cr;

% Convert body down (d) -> body z-up before applying DCM
B(3,:) = -B(3,:);

% Apply R to each column (vectorized elementwise)
Bx = B(1,:); By = B(2,:); Bz = B(3,:);

E_x = r11.*Bx + r12.*By + r13.*Bz;   % east
E_y = r21.*Bx + r22.*By + r23.*Bz;   % north
E_z = r31.*Bx + r32.*By + r33.*Bz;   % up

enu = [E_x; E_y; E_z];
end
```

Usage notes
- Provide Heading as geodetic heading (navigation convention). The function converts it internally.
- If you instead have Heading defined as math yaw (CCW from +x), call with deg=false and pass yaw directly or remove the psi conversion.
- The function is fully vectorized and returns ENU in meters (or same units as body).

Copilot:
Short answer
- There is no built-in frd2geodetic.  
- ecef2enu converts ECEF positions to local ENU positions (and ecef2enuv converts ECEF vectors to ENU vectors).  
- To go between FRD (body) and geodetic (lat,lon,h) you must combine an attitude-based body→ENU rotation with the Map Toolbox coordinate transforms (ENU↔ECEF or ENU↔geodetic).

Key distinctions
- FRD (forward,right,down) is a body‑fixed frame (vectors/positions relative to the vehicle, usually small offsets).
- ENU is a local tangent (east,north,up) Cartesian frame at a specified geodetic origin.
- ECEF is Earth‑centered Cartesian coordinates.
- Geodetic is (latitude, longitude, height).

Typical workflows (vectorized or single-sample):

1) Body vector → geodetic position offset
- Use your frd2enu (body→ENU) with the vehicle attitude to get a local ENU vector.
- Then convert ENU offset to geodetic coordinates (relative to a reference origin) with enu2geodetic.

Example (assume enu is small position offset from origin lat0,lon0,h0 and enu is [east; north; up]):

```matlab
% enu is 3x1 or 3xN
[lat, lon, h] = enu2geodetic(enu(1,:), enu(2,:), enu(3,:), lat0, lon0, h0, wgs84Ellipsoid);
```

2) Body vector → ECEF (position or vector)
- Convert body→ENU: enu = frd2enu(body, roll, pitch, heading, deg)
- Convert ENU→ECEF (positions): enu2ecef
  or convert ENU vector→ECEF vector: enu2ecefv (or use enu2ecef with correct vector handling)
- Alternatively, compute ECEF of origin and rotate: ECEF_point = enu2ecef(enu, lat0, lon0, h0, spheroid) + ecef_origin

3) ECEF → ENU or vector variants
- ecef2enu converts ECEF positions to ENU positions given origin.
- ecef2enuv converts ECEF vectors to ENU vectors (does not change origin).

Which built-in to use when
- You have geodetic lat/lon/h positions and want ENU: geodetic2enu.
- You have ECEF positions and want ENU: ecef2enu.
- You have ECEF vectors (velocity/displacement) and want ENU vectors: ecef2enuv.
- You have ENU and want geodetic: enu2geodetic.
- You have ENU and want ECEF: enu2ecef.

Minimal example: FRD point offset (body frame) at aircraft → geodetic point

```matlab
% Inputs:
% body = [f; r; d] (3x1) offset in body frame (meters)
% roll,pitch,Heading (Heading is geodetic nav heading: 0=north, cw)
% lat0,lon0,h0 = geodetic origin (vehicle position)
% sph = wgs84Ellipsoid;

% 1) body -> enu (use frd2enu that converts Heading->psi=90-Heading)
enu = frd2enu(body, roll, pitch, Heading, true);   % returns [east; north; up]

% 2) enu -> geodetic position (origin lat0,lon0,h0)
[lat, lon, h] = enu2geodetic(enu(1), enu(2), enu(3), lat0, lon0, h0, sph);
```

Notes and caveats
- enu2geodetic treats ENU as a position offset from the specified origin (not a unit direction). For small displacements this is fine; for large distances use care (nonlinear Earth curvature).
- Distinguish between converting a vector (direction/velocity) vs converting a point/position. Use the “v” variants (ecef2enuv, enu2ecefv) when you are transforming vectors.
- Ensure your heading convention: if Heading is geodetic (0° north, clockwise), convert to math yaw psi = 90° − Heading before building Rz; your frd2enu implementation should do this.
